# Contribution Guidelines

Thanks for considering a contribution! This list is for students learning to build real things. The bar is "would this help a student ship their first project?"

## What to add

Add an API if it meets ALL of these:

- ✅ **Free or has a generous free tier**
- ✅ **No credit card required to start** (or clearly marked if it is)
- ✅ **Documented** — students need to be able to use it without reverse-engineering
- ✅ **Active** — last update or response from the maintainer within 2 years
- ✅ **Reasonable rate limits** — enough for a student project to develop and demo

## What not to add

- ❌ APIs that require enterprise contracts to access
- ❌ Affiliate links or self-promotion
- ❌ Pay-only APIs with no free tier
- ❌ APIs that have been dead for years (e.g., deprecated endpoints)

## Format

In the category section, add a bullet in this format:

```
- 🆓 **[Name](https://api-url)** — One-sentence description of what it does and any notable limit.
```

Use the right legend symbol:

- 🆓 — no auth, no key
- 🔑 — free API key, no credit card
- 💳 — free tier exists but has a paid plan / credit card requirement
- 🏆 — recommended for first-time users (use sparingly, only for the best in each category)

Sort alphabetically within each category, with recommended (🏆) entries at the top.

## Adding code samples

If an API is commonly used, consider adding a code sample in the [Quick Start by Language](README.md#quick-start-by-language) section in multiple languages — at minimum JavaScript and one of PHP/Python.

## Adding project ideas

When you find a great API, propose a project idea using it in the [Project Ideas](README.md#project-ideas) section. Categorize by difficulty:

- **Weekend** — finishable in 1-2 days by a beginner
- **Semester** — group project / midterm scope
- **Capstone-Worthy** — real thesis or capstone potential

## Pull request

1. Check open issues and recent PRs to avoid duplicates.
2. One API per PR when possible.
3. Title: `Add [API Name] to [Category]` or `Update [API Name]`.
4. Briefly explain in the PR description why students would use it.

Thank you for helping students learn to build.
